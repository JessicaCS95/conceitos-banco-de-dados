Conceitos Banco de Dados

Normalização / Normas

Normalização é o processo de organização dos dados em um banco de dados para reduzir a redundância e melhorar a integridade dos dados. Esse processo divide as grandes tabelas em tabelas menores e estabelece relacionamentos entre elas, seguindo um conjunto de regras chamadas de formas normais. A principal vantagem da normalização é garantir que as atualizações, exclusões e inserções sejam feitas de maneira eficiente e sem inconsistências.

Redundância

Redundância em banco de dados refere-se à duplicação de dados. Ela pode ser intencional ou acidental. Em alguns casos, pequenas redundâncias podem ser toleráveis para melhorar o desempenho da consulta, mas o excesso de redundância pode levar a inconsistências e a um maior espaço de armazenamento. A normalização, como mencionado anteriormente, busca minimizar essa duplicação.

Concorrência

Concorrência ocorre quando múltiplas operações tentam acessar ou modificar dados ao mesmo tempo. Em bancos de dados, isso pode causar conflitos ou inconsistências, especialmente quando diferentes transações tentam modificar o mesmo dado simultaneamente. Para resolver esses problemas, os sistemas de gerenciamento de banco de dados implementam controle de concorrência, como bloqueios (locks) ou versionamento.

Integridade

A integridade de dados refere-se à manutenção da precisão e consistência das informações durante todo o seu ciclo de vida, garantindo que os dados sejam gravados e armazenados de forma confiável para que possam ser recuperados mesmo diante de falhas como erros humanos ou defeitos de hardware. Existem dois tipos de integridade de dados:

  • Integridade Física: Trata dos desafios relacionados ao armazenamento, como falhas mecânicas, desastres naturais e quedas de energia. Medidas como redundância de hardware e migração para a nuvem ajudam a proteger os dados.
  • Integridade Lógica: Protege os dados em bancos relacionais contra erros e ataques. Ela se divide em:
    • Integridade da Entidade: Garante que não haja campos nulos ou duplicados nas tabelas.
    • Integridade Referencial: Mantém a consistência dos dados por meio de chaves de acesso e restrições.
    • Integridade de Domínio: Assegura que os valores em cada coluna sigam regras específicas de formato e tipo.
    • Integridade Definida pelo Usuário: Permite a criação de regras personalizadas para atender necessidades específicas.

Criptografia

Criptografia é a técnica de proteger os dados convertendo-os em um formato ilegível para aqueles que não possuem a chave de decodificação. Em bancos de dados, a criptografia pode ser usada para proteger informações sensíveis tanto em repouso (dados armazenados) quanto em trânsito (dados sendo transmitidos entre servidores e clientes).

Consistência

Consistência garante que, após qualquer operação de banco de dados, o sistema sempre estará em um estado válido. Por exemplo, uma transação que quebra uma regra de integridade do banco de dados será revertida para garantir que o sistema permaneça consistente. O conceito está fortemente relacionado à integridade e faz parte do conjunto de propriedades ACID (Atomicidade, Consistência, Isolamento, Durabilidade) que regem as transações em bancos de dados.

Autenticação

Autenticação refere-se ao processo de verificar a identidade de um usuário ou sistema antes de permitir o acesso ao banco de dados. Esse processo geralmente envolve a inserção de um nome de usuário e senha, mas pode incluir fatores adicionais, como autenticação multifator (MFA), para aumentar a segurança.

SQL Injection

SQL Injection é uma técnica de ataque em que um invasor insere código malicioso em uma consulta SQL para manipular ou roubar informações do banco de dados. Esse tipo de ataque é comum em aplicativos que não validam adequadamente as entradas dos usuários e é uma ameaça grave à segurança do banco de dados.

Ameaças

Ameaças em bancos de dados incluem qualquer risco que possa comprometer a integridade, a confidencialidade ou a disponibilidade dos dados. Exemplos de ameaças incluem SQL Injection, acesso não autorizado, malwares, falhas de hardware e violações de dados.

Controle de Acesso

Controle de acesso refere-se à prática de garantir que apenas usuários autorizados possam acessar ou modificar dados em um banco de dados. Os sistemas de controle de acesso podem usar diferentes níveis de permissões, como leitura, escrita e exclusão, para gerenciar o que cada usuário pode fazer.

Cópia de Segurança (Backup)

Cópia de segurança (ou backup) é a criação de uma cópia dos dados armazenados em um banco de dados, com o objetivo de garantir que eles possam ser restaurados em caso de falha no sistema, exclusão acidental ou ataque malicioso.

Restore

Restore é o processo de recuperação dos dados a partir de uma cópia de segurança. Em caso de falha ou corrupção de dados, o restore é realizado para restaurar o banco de dados ao seu estado anterior.

Linguagens

Linguagem de consulta estruturada (Structured Query Language - SQL)

SQL é a linguagem padrão para gerenciar e manipular bancos de dados relacionais. Ela é usada para realizar operações como inserção, consulta, atualização e exclusão de dados, além de gerenciar a estrutura do banco de dados.

Linguagem de definição de dados (Data Definition Language - DDL)

DDL é o subconjunto de SQL usado para definir e gerenciar a estrutura de um banco de dados, como criar, alterar ou excluir tabelas. Exemplos de comandos DDL incluem CREATE, ALTER e DROP.

Linguagem de manipulação de dados (Data Manipulation Language - DML)

DML é o conjunto de comandos SQL usados para manipular dados dentro de tabelas. Ele inclui operações como inserir, atualizar, excluir e consultar dados. Exemplos de comandos DML incluem INSERT, UPDATE, DELETE e SELECT.

Linguagem de consulta de dados (Data Query Language - DQL)

DQL é o subconjunto de SQL que trata exclusivamente da consulta de dados. O comando principal de DQL é o SELECT, que permite a recuperação de informações do banco de dados.
